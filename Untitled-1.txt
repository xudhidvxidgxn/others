데이터 -> 단순 수집한 사실 / 값 

정보 -> 의사 결정 활용할 수 있게 데이터 처리

정보 처리 -> 데이터 정보 추출 과정/방법

데이터베이스 -> 
정보 얻기 위해 데이터 모아 구조적 통합
조직의 사용자가 공유 사용할 수 있게 통합 저장 운영 데이터 집합

데이터베이스 장점 -> 
중복 데이터 최소화
데이터 일관성 유지, 공동 사용
데이터 무결성, 기밀성 유지
데이터의 물리적, 논리적 독립성 유지

데이터베이스 단점 -> 
운영비 증대
자료처리 방법 복잡
백업, 복구 어려움

릴레이션 -> 표 (테이블)
속성 -> 릴레이션 구성 열 (세로줄) 
튜플 -> 릴레이션 구성 행 (가로줄), (= 레코드)
도메인 -> 속성이 가질 수 있는 값의 범위 (성별 도메인 남자/여자)
릴레이션 스키마 -> 제목줄(= 헤더)
릴레이션 인스턴스 -> 제목줄을 제외하고 입력된 내용(= 바디)
차수 -> 릴레이션 구성 속성 수 (= 세로행 수)
카디널리티 -> 릴레이션 튜플 수 (= 가로행 수), 제목줄 제외

릴레이션 특성 
튜플의 유일성 -> 릴레이션에 동일 튜플 존재 불가 
    [표에서 동일한 행이 존재하지 못함]
튜플의 무순서 -> 릴레이션에 튜플 사이 순서 무의미 (행) 
    [표에서 행간 순서 무의미]
속성의 무순서 -> 릴레이션에 속성 사이 순서 무의미 (열) 
    [표에서 열간 순서 무의미]
속성의 원자성 -> 속성 값 원자 값 
    [속성에는 단일 값밖에 들어가지 못합]

데이터베이스 용어
DDL : Database Defination Language
    - 테이블 생성 : CREATE TABLE
    - 테이블 변경 : ALTER TABLE
    - 테이블 삭제 : DELETE TABLE
DML : Database Manipulation Language 
    - 데이터 검색 : SELECT 
    - 데이터 삽입 : INSERT
    - 데이터 수정 : UPDATE 
    - 데이터 삭제 : DELETE 
DCL : Database Control Language 
    - 사용자 권한 부여 : GRANT
    - 사용자 권한 제거 : REVOKE
TCL : Transaction Control Language 
    - 실행 결과 반영 : COMMIT 
    - 원래 상태로 복귀 : ROLLBACK

SQL(Structed QUery Language) 
    - 사용자와 데이터베이스 관리 시스템 간의 통신 수단 
    - 사용 목적에 따라 데이터 정의어, 데이터 조작어, 데이터 제어어로 구분
        1. 데이터 정의어 (DDL) : 스키마 정의 수정 삭제
        2. 데이터 조작어 (DML) : 데이터 삽입 삭제 수정 검색 등
        3. 데이터 제어어 (DCL) : 데이터베이스에 접근 권한 부여 제거
        4. 트랜잭션 제어어 (TCL) : DML로 실행한 변경 사항을 반영/복귀
